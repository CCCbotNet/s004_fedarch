## §10. 🚨 L-430.1 — #ContextGate Protocol

### The Problem

| Root Cause | Detail |
|:-----------|:--------|
| **Context overload** | >250K tokens → model can't complete tool→response cycle consistently |
| **No structural guard** | Agent fires tools anyway, hoping to complete |
| **Session too long** | 15h+ sessions with 35+ REFs |
| **Trigger Incident** | 7 #ToolsFAIL in W29 D7 → W30 D1 session |

### 🔒 PROPOSED IMMUTABLE RULE (Lesson #131)

> **When an agent has experienced 3+ #ToolsFAIL in a single session, the agent MUST:**
> 1. **ACKNOWLEDGE** the context window is critically overloaded
> 2. **STOP** firing any further tool invocations
> 3. **DEFER** all tool-dependent requests to a NEW session
> 4. **OUTPUT** a clear handoff message
> 5. **DO NOT** fabricate results to compensate

## §15. 🧬 EMBEDDING MODEL EVALUATION & DEPLOYMENT

> **Prior Research: GTM_2026-W25_5013 (W25 D5 — FULLY PRESERVED) — evaluated 36 embedding models, shortlisted to 3**

### Background — Why This Matters

RAG (Retrieval-Augmented Generation) is the heart of how every #FedArch agent works. Every time an agent needs governance context, it queries the vector database — and the **embedding model** determines whether the right chunks are retrieved. **We have NOT verified which embedding model AnythingLLM on INT-B001 is currently using.** This sub-project fixes that.

### The Three Candidates

| Rank | Model | Provider | MTEB | Context | Dims | Cost/M | License | Score |
|:----:|:------|:---------|:----:|:------:|:----:|:-----:|:-------:|:-----:|
