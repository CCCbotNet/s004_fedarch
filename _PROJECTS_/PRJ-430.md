## §9. 🚨 L-430.1 — #ContextGate Protocol

### The Problem

| Root Cause | Detail |
|:-----------|:--------|
| **Context overload** | >250K tokens → model can't complete tool→response cycle |
| **No structural guard** | Agent fires tools anyway, hoping to complete |
| **Session too long** | 15h+ sessions with 35+ REFs |
| **Trigger Incident** | 7 #ToolsFAIL in W29 D7 → W30 D1 session |

### 🔒 PROPOSED IMMUTABLE RULE (Lesson #131)

> **When an agent has experienced 3+ #ToolsFAIL in a single session, the agent MUST:**
> 1. **ACKNOWLEDGE** the context window is critically overloaded
> 2. **STOP** firing any further tool invocations
> 3. **DEFER** all tool-dependent requests to a NEW session
> 4. **OUTPUT** a clear handoff message
> 5. **DO NOT** fabricate results to compensate

### Session Lifecycle

| Stage | Duration | Context Health | Action |
|:------|:--------:|:--------------:|:-------|
| 🟢 **GREEN** | 0-4h | ✅ Excellent | Normal operations |
| 🟡 **YELLOW** | 4-8h | ⚠️ Moderate | Monitor. Consider closing. |
| 🟠 **ORANGE** | 8-12h | 🔴 Heavy | Preemptive warning. Plan close. |
| 🔴 **RED** | 12h+ | 🔴 Critical | #ContextGate may engage. Close immediately. |

### Thresholds

| #ToolsFAIL | Action |
|:----------:|:-------|
| 0 | 🟢 Normal |
| 1 | 🟡 Note |
| 2 | 🟠 Warning |
| **3+** | 🔴 **#ContextGate ENGAGED** |
