## §2. 📋 What Is Pushover?

### The Product

Pushover is a **simple notification service** that enables sending push notifications from any application, service, or script to mobile devices and desktops. It was founded in 2012 by Pushover, LLC and has been operating for 14+ years.

### Supported Platforms

| Platform | App | Cost |
|:---------|:----|:----:|
| **Android** | ✅ Google Play | $4.99 one-time (30-day free trial) |
| **iPhone & iPad** | ✅ App Store | $4.99 one-time (30-day free trial) |
| **Desktop (Browser)** | ✅ Web app | $4.99 one-time (30-day free trial) |
| **Android Wear** | ✅ Included | Covered by Android purchase |
| **Apple Watch** | ✅ Included | Covered by iOS purchase |
| **Multiple devices (same platform)** | ✅ All covered | One purchase covers all |

### Key Differentiator

> **Pushover's defining feature is SIMPLICITY.** It does ONE thing — push notifications — and does it extremely well. No chat. No collaboration tools. No dashboards. Just: trigger → notification → recipient receives it.

## §4. 🔧 How It Works

### The API (Simple HTTP POST)

Pushover's API is famously simple. A single curl command sends a notification:

```bash
curl -s \
  --form-string "token=abc123" \
  --form-string "user=user123" \
  --form-string "message=hello world" \
  https://api.pushover.net/1/messages.json
```

### Supported Languages (Native Examples)

| Language | Lines of Code |
|:---------|:-------------:|
| **curl** | 1 command |
| **Python 3** | 5 lines |
| **Ruby** | 8 lines |
| **Go** | 12 lines |
| **Perl** | 4 lines |
| **PHP** | 8 lines |
| **Node.js** | Via npm package |
| **Any HTTP library** | Works with every language |

### Additional Delivery Methods

| Method | How It Works |
|:-------|:-------------|
| **Email Gateway** | Send email to `userkey@pomail.net` → converts to push notification |
| **IFTTT** | Pushover channel — connect 1000+ services |
| **Zapier** | Pushover integration — 4000+ app connections |
| **Webhook** | GitHub, GitLab, and any webhook-capable service |

### Notification Features

| Feature | Details |
|:--------|:--------|
| **Title** | Optional message title (default: app name) |
| **Message** | Up to 1024 characters |
| **Priority** | Normal, High, Emergency (requires acknowledgment) |
| **Sound** | Choose from 30+ notification sounds |
| **Timestamp** | Custom timestamp support |
| **URL** | Optional URL to open on tap |
| **URL Title** | Custom title for attached URL |
| **Device targeting** | Send to specific devices, not all |
| **HTML support** | Basic HTML formatting in messages |

## §8. ✅❌ Pros & Cons

### Pros

| # | Pro | Impact |
|:-:|:----|:-------|
| 1 | **Insanely simple API** — 1 curl command to send notifications | Any dev in the ecosystem can integrate in minutes |
| 2 | **One-time purchase** for individuals — $4.99/platform | No subscription fatigue for agency owners |
| 3 | **14+ years operational** — 2012–2026 | Not a fly-by-night service. Established and stable. |
| 4 | **Native Prometheus Alertmanager integration** | Direct pipeline from our SigNoz setup |
| 5 | **No account required to receive** (only to send) | Agency owners just install app and provide user key |
| 6 | **Cross-platform** — Android, iOS, iPad, Desktop, Watch | Everyone covered regardless of device preference |
| 7 | **Email gateway** — `userkey@pomail.net` | Can trigger notifications from any system that can send email |
| 8 | **70+ integrations** — WordPress, GitHub, Zabbix, Sentry | Already covers our tech stack |
| 9 | **No data mining. No ads. No tracking.** | Aligned with our privacy values |
| 10 | **Emergency priority** — requires acknowledgment | Critical alerts cannot be ignored |

### Cons

| # | Con | Mitigation |
|:-:|:----|:-----------|
| 1 | **Not self-hostable** — SaaS only | For INDEP Agency owners, this is actually a PRO (no infrastructure). For core infra, we can still use Gotify alongside. |
| 2 | **US-based company** (Pushover, LLC) | WeOwnNet is US-based ecosystem. Acceptable. Swiss/European privacy advocates may prefer Proton ecosystem. |
| 3 | **$5/user/mo Teams** if centralized management needed | Individual licenses ($4.99 one-time) work for most agency owners. Teams only needed for centralized management. |
| 4 | **10K message/month limit on individual** | 10K = ~333 messages/day. For alerting, this is ample. Only high-volume systems need upgrades. |
| 5 | **No end-to-end encryption** | Notifications are not E2EE. Acceptable for operational alerts. For sensitive data, use different channel. |
| 6 | **Limited message length** (1024 chars) | Enough for alert summaries. Reference links provide full detail. |
