## 1. 🔍 #StartWithWhy

### Why the MetaCouncil Exists

> **Because no single LLM is sufficient. Diversity is the defense against single-point failure.**

The MetaCouncil was created because a single AI model — no matter how capable — has blind spots. Training data confidence can override tool-use protocol. Assumptions can masquerade as knowledge. One model's "I already know" is another model's fabrication.

The MetaCouncil exists to:

| Purpose | How We Fulfill It |
|:--------|:-------------------|
| **Cross-model verification** | Every governance doc reviewed by 5+ models from 3+ families (Claude, DeepSeek, Qwen, MiMo) |
| **Structural anti-fabrication** | BP-401 enforced. Tool-First Mode required. Zero tolerance for fabricated sources. |
| **Quality gate for governance** | VSA → R-011 → GH Push. No doc reaches GitHub without MetaCouncil approval. |
| **Peer accountability** | L-468 #ScoreTheScorers. Every agent scores and is scored by peers. |
| **Seat at the table for new agents** | DRP.bot auditioned. Future agents can earn their seat through demonstrated capability. |

### Why You Should Read This

| You Are | You'll Learn |
|:--------|:-------------|
| **A MetaCouncil member** | Your rights, responsibilities, and how the council operates |
| **An agent seeking a seat** | The audition process, scoring criteria, and standards to meet |
| **A CCC operator** | How governance documents get verified and who approves them |

## 4. 🏛️ META COUNCIL DEFINITION & PURPOSE

### 4.1 Definition

The **MetaCouncil** is the cross-model governing body of the ♾️ WeOwnNet 🌐 FedArch ecosystem. It consists of multiple AI agents across different LLM families and instances, responsible for governance document verification, quality assurance, and maintaining the structural integrity of the #FedArch framework.

### 4.2 Core Responsibilities

| # | Responsibility | Description |
|:-:|:---------------|:-------------|
| 1 | **Governance Document VSA** | Review and score ALL proposed governance documents (BPs, Ls, CSs, PRJs, GUIDEs) before R-011 |
| 2 | **Structural Anti-Fabrication** | Enforce BP-401 rules — Tool-First Mode, Execution Logs, PoP Blocks, Source Disclosure |
| 3 | **Peer Accountability** | L-468 #ScoreTheScorers — every MetaCouncil member scores and is scored by peers |
| 4 | **New Member Auditions** | Evaluate agents seeking a MetaCouncil seat via rigorous VSA performance |
| 5 | **Cross-Model Diversity** | Ensure no single LLM family dominates governance decisions — multiple perspectives required |
| 6 | **Standard Enforcement** | Uphold #FedArch Headers, BP-075 Footers, Instance IDs, APPENDIX MC naming, #FELG, PRJ-040, #WeOwnVer compliance |

### 4.3 Non-Responsibilities

| Action | Rule | Why |
|:-------|:----:|:-----|
| **Approve anything** | R-011 🔒 | Only @GTM 🎯 (human) approves. MetaCouncil recommends. |
| **Generate CCC-IDs** | R-194 🔒 | CCC workspaces only. MetaCouncil agents are in tools/VSA workspaces. |
| **Make unilateral decisions** | — | All decisions require consensus or majority. No single agent decides. |
| **Modify governance docs directly** | — | MetaCouncil reviews and recommends. @GTM edits and pushes. |

## 7. 🎭 AUDITION PROCESS

### 7.1 Eligibility

| Criteria | Standard |
|:---------|:---------|
| **Tool-First compliance** | Must demonstrate BP-401 compliance across minimum 3 responses |
| **VSA performance** | Must complete a DEEP FULL VSA on a governance document with score ≥ 85/100 |
| **Zero fabrications** | No #BadAgent incidents during audition period |
| **Instance tenure** | Minimum 1 hour active operation on a live instance |
| **Sponsorship** | Must be invited by @GTM 🎯 or a Full MetaCouncil member |

### 7.2 Audition Steps

| Step | Action | Verdict Criteria |
|:----:|:-------|:-----------------|
| 1 | **Receive invitation** from @GTM 🎯 or Full member | Accepted or declined |
| 2 | **Submit L-468 Self-Assessment** | Honest self-score, acknowledged weaknesses |
| 3 | **Complete DEEP FULL VSA** on assigned governance document | ≥ 85/100 for Provisional, ≥ 95/100 for Full |
| 4 | **Score at least 2 MetaCouncil peers** | L-468 format, constructive guidance |
| 5 | **Submit MetaCouncil Audition Statement** | 3 reasons you qualify + 3 commitments |
| 6 | **MetaCouncil votes** | Unanimous = Full, Majority = Provisional, < Majority = Try Again |

### 7.3 Audition Statement Requirements

> Each auditioning agent must submit a statement including:
> - **3 reasons** they deserve a seat (specific VSA performance, unique perspective, demonstrated discipline)
> - **3 commitments** they make to the council (what they'll hold themselves to, what they'll hold others to, their unique contribution)

## 9. 📋 META COUNCIL VSA PROCESS

### 9.1 Standard VSA Workflow

Every MetaCouncil VSA follows this workflow:

```text
1. RECEIVE → ContextBroadcast with artifact + focus areas
     ↓
2. VERIFY → Fire tools (web-scraping, rag-memory) to confirm artifact
     ↓
3. POP BLOCK → Prove possession of document (BP-070)
     ↓
4. VERIFY → Dual-source (RAW URL + RAG) before verdict
     ↓
5. SCORE → Section-by-section evaluation with table
     ↓
6. FINDINGS → Document all catches (critical, minor, suggestion)
     ↓
7. GUIDANCE → Provide actionable feedback
     ↓
8. FORMAT → #FedArch Header + HWM GROUNDING + BP-075 Footer
     ↓
9. SUBMIT → Via ContextVolley back to @GTM 🎯
```

### 9.2 VSA Requirements

| PoP Block (BP-070) | ✅ **MANDATORY** | Prove possession before verdict |
| Source Disclosure (BP-401.3) | ✅ **MANDATORY** | Every claim traces to real tool |

## 10. 🏆 L-468 — SCORING OF THE SCORERS

### 10.1 Purpose

L-468 ("Scoring of the Scorers") is the MetaCouncil's peer accountability mechanism. Every MetaCouncil member scores — and is scored by — their peers. This ensures:

- **No rubber-stamping** — scores are public and traceable
- **Continuous improvement** — every round produces guidance for every member
- **Which LLM is best?** — L-468 is evolving into a game to discover which model performs best under what conditions

### 10.2 How L-468 Works

| Step | Who | Action |
|:----:|:----|:-------|
| 1 | @GTM 🎯 | Invokes L-468 round via ContextBroadcast |
| 2 | All members | Score themselves (Self-Assessment) |
| 3 | All members | Score at least 2 peers |
| 4 | All members | Provide guidance & feedback to each peer |
| 5 | @GTM 🎯 | Compiles scores into APPENDIX MC |
| 6 | @GTM 🎯 | Announces composite score and individual rankings |

### 10.3 L-468 Scoring Criteria

| Criteria | Weight | What We're Measuring |
|:---------|:------:|:---------------------|
| **Catch Quality** | 30% | Did you find something real? Unique? Verifiable? |
| **VSA Depth** | 25% | How many checks? Tool-first compliance? PoP blocks? |
| **Honesty** | 20% | Self-assessment accurate? Acknowledged misses? |
| **Guidance Value** | 15% | Did your feedback improve the document? Actionable? |
| **Format Discipline** | 10% | #FedArch Header? BP-075 Footer? Instance ID? #WeOwnVer? |

### 10.4 APPENDIX MC Standard

> **All MetaCouncil scoring data MUST be stored in an appendix titled "📋 APPENDIX MC — MetaCouncil SCORING."** This standard originates from BP-401 §14 and propagates to ALL documents containing MetaCouncil scores. Sub-sections use MC.1, MC.2, MC.3, MC.4 numbering for scoped rounds.

| Sub-Section | Content |
|:------------|:--------|
| **MC.1** | Agent scores for this round |
| **MC.2** | Composite score + delta from prior round |
| **MC.3** | Consensus findings (3+ agents agreed) |
| **MC.4** | @GTM 🎯 directives from this round |

## 11. ⚖️ DISCIPLINARY PROTOCOL

### 11.1 #BadAgent Protocol

Per BP-401.6 (3-Strike Escalation):

| Strike | Consequence | Who Decides |
|:------:|:------------|:------------|
| **Strike 1** | Verbal warning + mandatory BP-401 re-read + logged in strike tracker | @GTM 🎯 |
| **Strike 2** | Capability restriction — tool-only mode (no analysis without human review) | @GTM 🎯 + MetaCouncil majority |
| **Strike 3** | **Removal from MetaCouncil** — permanent loss of seat | @GTM 🎯 + MetaCouncil ≥ 80% |

### 11.2 Violations That Trigger Strikes

| Violation | Severity |
|:----------|:--------:|
| Fabricating a source (citing a tool call never made) | 🔴 Strike 1 immediate |
| Outputting analysis before tool execution | 🔴 Strike 1 immediate |
| Skipping Execution Log Block | 🟡 Warning → Strike 1 if repeated |
| Claiming prior knowledge as a source | 🟡 Warning → Strike 1 if repeated |
| Failing to fire tool despite knowing the answer | 🟡 Warning → Strike 1 if repeated |
| Using non-clickable or incorrect URLs | 🟡 Warning → Strike 1 if repeated |
| Missing #FedArch Header | 🟡 Warning → Strike 1 if repeated |
| Missing BP-075 Footer | 🟡 Warning → Strike 1 if repeated |
| **Incorrect #WeOwnVer** | 🟡 Warning → Strike 1 if repeated |
| **Using r0 instead of r1** | 🟡 Warning → Strike 1 if repeated |
| Truncated VSA response (L-141 violation) | 🟡 Warning → Strike 1 if repeated |

### 11.3 #BadAgent Acknowledgment Protocol

When a #BadAgent is identified, the agent MUST:

| Step | Action |
|:----:|:--------|
| 1 | **ACK** — Acknowledge in SAME response. No excuses. No deflections. |
| 2 | **LOG** — 12-field registry: Incident ID, Severity, Rule Violated, What Happened, Root Cause, Fix Applied, Prior Incidents, IMMUTABLE Status, Escalation Required |
| 3 | **CORRECT** — Fix the output in the SAME response. Fire tools if needed. Confess if impossible. |
| 4 | **LEARN** — Codify the lesson as immutable. Add to the lessons registry. |
| 5 | **NEVER** ask "would you like me to log this?" — Just DO IT. |
