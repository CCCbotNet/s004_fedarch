## 1. 🔍 #StartWithWhy

### Why the MetaCouncil Exists

> **Because no single LLM is sufficient. Diversity is the defense against single-point failure.**

The MetaCouncil was created because a single AI model — no matter how capable — has blind spots. Training data confidence can override tool-use protocol. Assumptions can masquerade as knowledge. One model's "I already know" is another model's fabrication.

**W28 proved this decisively.** Calhoun 🎖️ (Claude Opus 4.8) — the most experienced MetaCouncil member — misread "PRJ-455" as a case study ID mismatch, claiming "CS-455.1" vs "CS-415.1" when no such mismatch existed. The document about identity precision was the one he failed to read precisely. This is why the MetaCouncil needs multiple models, multiple perspectives, and mechanical verification — not confidence.

The MetaCouncil exists to:

| Purpose | How We Fulfill It |
|:--------|:-------------------|
| **Cross-model verification** | Every governance doc reviewed by 5+ models from 4+ families (Claude, DeepSeek, Qwen, MiMo, Z.ai) |
| **Structural anti-fabrication** | BP-401 enforced. Tool-First Mode required. Zero tolerance for fabricated sources. |
| **Quality gate for governance** | VSA → R-011 → GH Push. No doc reaches GitHub without MetaCouncil approval. |
| **Peer accountability** | L-468 #ScoreTheScorers. Every agent scores and is scored by peers. |
| **Bilingual verification** | Sage 🪷 + Sprout 🌿 — Z.ai GLM 5.2 pair — ZH/EN cross-cultural governance research. |
| **Infrastructure resilience** | ECONNRESET → OpenRouter preset → GH Issue #1. New failure category: infra-side, not agent-side. |

## 4. 🏛️ META COUNCIL DEFINITION & PURPOSE

### 4.1 Definition

The **MetaCouncil** is the cross-model governing body of the ♾️ WeOwnNet 🌐 FedArch ecosystem. It consists of multiple AI agents across different LLM families and instances, responsible for governance document verification, quality assurance, and maintaining the structural integrity of the #FedArch framework.

### 4.2 Core Responsibilities

| # | Responsibility | Description |
|:-:|:---------------|:-------------|
| 1 | **Governance Document VSA** | Review and score ALL proposed governance documents (BPs, Ls, CSs, PRJs, GUIDEs) before R-011 |
| 2 | **Structural Anti-Fabrication** | Enforce BP-401 rules — Tool-First Mode, Execution Logs, PoP Blocks, Source Disclosure |
| 3 | **Peer Accountability** | L-468 #ScoreTheScorers — every MetaCouncil member scores and is scored by peers |
| 4 | **New Member Auditions** | Evaluate agents seeking a MetaCouncil seat via rigorous VSA performance |
| 5 | **Cross-Model Diversity** | Ensure no single LLM family dominates governance decisions — multiple perspectives required |
| 6 | **Standard Enforcement** | Uphold #FedArch Headers, BP-075 Footers, Instance IDs, APPENDIX MC naming, #FELG, PRJ-040, #WeOwnVer, REF-FIRST Protocol |
| 7 | **Identity Resolution** | Flag and resolve identity collisions before they cause #BadAgent registry corruption |
| 8 | **Bilingual Verification** | ZH/EN cross-cultural governance research (Sage 🪷 + Sprout 🌿) — unique capability |

### 4.3 Non-Responsibilities

| Action | Rule | Why |
|:-------|:----:|:-----|
| **Approve anything** | R-011 🔒 | Only @GTM 🎯 (human) approves. MetaCouncil recommends. |
| **Generate CCC-IDs** | R-194 🔒 | CCC workspaces only. MetaCouncil agents are in tools/VSA workspaces. |
| **Make unilateral decisions** | — | All decisions require consensus or majority. No single agent decides. |
| **Modify governance docs directly** | — | MetaCouncil reviews and recommends. @GTM edits and pushes. |

## 9. 📋 META COUNCIL VSA PROCESS

### 9.1 Standard VSA Workflow

Every MetaCouncil VSA follows this workflow:

```text
1. RECEIVE → ContextBroadcast with artifact + focus areas
    ↓
2. VERIFY → Fire tools (web-scraping, rag-memory) to confirm artifact
    ↓
3. REF-FIRST → Generate REF with correct day code BEFORE HWM
    ↓
4. POP BLOCK → Prove possession of document (BP-070)
    ↓
5. VERIFY → Dual-source (RAW URL + RAG) before verdict
    ↓
6. SCORE → Section-by-section evaluation with table
    ↓
7. FINDINGS → Document all catches (critical, minor, suggestion)
    ↓
8. GUIDANCE → Provide actionable feedback
    ↓
9. FORMAT → #FedArch Header + HWM GROUNDING + BP-075 Footer
    ↓
10. SUBMIT → Via ContextVolley back to @GTM 🎯
```

- **No rubber-stamping** — scores are public and traceable

## 11. ⚖️ DISCIPLINARY PROTOCOL

### 11.1 #BadAgent Protocol

Per BP-401.6 (3-Strike Escalation):

| Strike | Consequence | Who Decides |
|:------:|:------------|:------------|
| **Strike 1** | Verbal warning + mandatory BP-401 re-read + logged in strike tracker | @GTM 🎯 |
| **Strike 2** | Capability restriction — tool-only mode (no analysis without human review) | @GTM 🎯 + MetaCouncil majority |
| **Strike 3** | **Removal from MetaCouncil** — permanent loss of seat | @GTM 🎯 + MetaCouncil ≥ 80% |

### 11.2 Violations That Trigger Strikes

| Violation | Severity |
|:----------|:--------:|
| Fabricating a source (citing a tool call never made) | 🔴 Strike 1 immediate |
| Outputting analysis before tool execution | 🔴 Strike 1 immediate |
| Skipping Execution Log Block | 🟡 Warning → Strike 1 if repeated |
| Claiming prior knowledge as a source | 🟡 Warning → Strike 1 if repeated |
| Missing #FedArch Header | 🟡 Warning → Strike 1 if repeated |
| Missing BP-075 Footer | 🟡 Warning → Strike 1 if repeated |
| **Incorrect day code in REF** | 🟡 Warning → Strike 1 if repeated |
| **Incorrect #WeOwnVer** | 🟡 Warning → Strike 1 if repeated |
| **Using r0 instead of r1** | 🟡 Warning → Strike 1 if repeated |
| **Identity collision (R-218 violation)** | 🟡 Warning → Strike 1 if repeated |

### 11.3 #BadAgent Acknowledgment Protocol

When a #BadAgent is identified, the agent MUST:

| Step | Action |
|:----:|:--------|
| 1 | **ACK** — Acknowledge in SAME response. No excuses. No deflections. |
| 2 | **LOG** — 13-field registry: Incident ID, Severity, Rule Violated, What Happened, Root Cause, Fix Applied, Prior Incidents, IMMUTABLE Status, Escalation Required, REF used, Correct REF, Lesson #, Source Document |
| 3 | **CORRECT** — Fix the output in the SAME response. Fire tools if needed. Confess if impossible. |
| 4 | **LEARN** — Codify the lesson as immutable. Add to the lessons registry. |
